# PhonePe: Payment Insights Dashboard (Internet Dataset Analysis)

A comprehensive data engineering and visualization project designed to analyze corporate digital transaction patterns. This dashboard leverages an open internet dataset to model, transform, and present key metrics that power digital payments across Bharat, mirroring the core interface guidelines of the PhonePe ecosystem.

---

## 📊 Dataset Context & Origin
* **Source:** Open-source internet transaction dataset (simulated financial records modeling digital payment behavior across localized markets).
* **Scope:** 12 months of longitudinal transaction data covering demographic metrics, channel failures, user values, and multi-service distributions.
* **Volume:** Aggregated metrics handling over 300,000 unique records across multiple service nodes.

---

## 🚀 Key Features & Performance Metrics

### 1. High-Level KPI Scorecard
* **Total Transactions:** 300K processed transactions (demonstrating an 8.97% growth month-over-month).
* **Total Transaction Value:** ₹3.47 Billion in volume managed (growing at 8.98% per cycle).
* **Active User Base:** 108K unique active digital consumers interacting with the platform.
* **System Success Rate:** High operational reliability maintained at a healthy 96.00%.

### 2. Deep-Dive Analytical Visualizations
* **Temporal Trends (Transactions Over Time):** A dual-axis line chart mapping seasonal volume trends against gross transaction values across all 12 fiscal months.
* **Demographic Segment Contribution:** A donut chart highlighting generation-based contributions, with Gen X (37.4%) and Millennials (37.3%) driving the bulk of transactions.
* **Vertical Service Analysis:** A horizontal bar chart identifying **Money Transfers (0.15M)** as the primary driver compared to peripheral services like insurance, loans, and bill recharges.
* **High-Value User Profiling:** Ranked tracking of the Top 5 Users sorted by gross transaction values.
* **Temporal Split Analysis:** Behavior mapping indicating heavy platform reliance during weekdays (71.6% / 215K transactions) vs. weekends (28.4% / 85K transactions).

---

## 🛠️ System Architecture & Interactive Controls
* **Global Filter Matrix:** Allows real-time dataset filtering based on specific calendar months (e.g., January) and isolated execution flags (`Failed`, `Pending`, `Successful`).
* **Visual Identity:** Built with high UI scannability adhering to PhonePe's traditional design palette (deep indigos, violet accents, and sharp contrasting white space).

---

## 🎯 Strategic Project Goals
1. Transform raw, unorganized internet datasets into clean, actionable relational metrics.
2. Evaluate system load distribution to spot peak transaction hours and weekday vs. weekend shifts.
3. Deliver a production-ready analytical tool that mimics corporate-grade payment oversight structures.
